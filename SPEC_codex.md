# SPEC_codex

## 목표 사용자

- Blue Verse를 처음 보는 잠재 고객
- 복잡한 운영 구조를 가진 서비스 구축 파트너를 찾는 의사결정자
- 사례를 빠르게 훑고 신뢰 판단을 내리고 싶은 방문자

## 사용자 스토리

- 방문자는 첫 화면에서 이 팀이 "무슨 분야"보다 "어떤 복잡도"에 강한지 바로 이해해야 한다.
- 방문자는 대표 사례 3건과 전체 라이브러리를 볼 때 카드가 흐트러져 보이지 않고, 정보 위계가 또렷해야 한다.
- 방문자는 전체 프로젝트 목록에서 13개 사례를 문제 유형 기준으로 부담 없이 비교할 수 있어야 한다.
- 방문자는 About 페이지에서 팀의 태도와 작업 방식을 더 차분하게 확인할 수 있어야 한다.

## 핵심 문제

- 현재 Featured와 Library 카드가 여전히 넓고 느슨해 보여 정리된 인상보다 어수선한 인상을 준다.
- 카드 라운드와 패딩이 여전히 커서 작은 카드 수와 맞지 않는 비율이 남아 있다.
- 헤더 워드마크가 직전 패스 기준으로는 과하게 커 본문 진입 전에 무게중심이 위로 쏠린다.
- 일부 카피가 아직 `12개 사례` 기준으로 남아 있어 실제 13개 프로젝트 데이터셋과 맞지 않는다.

## 리디자인 요구사항

### 정보구조

#### index.html

1. Quiet Hero
2. Featured Cases
3. Project Library
4. Compact Method Strip
5. CTA / Footer

#### about.html

1. About Hero
2. Positioning Statement
3. Process
4. Coverage
5. CTA / Footer

### 헤더 요구사항

- 헤더 로고는 실제 PNG 워드마크를 사용한다
- HTML 이미지 속성에는 실제 자산 치수(593x117)를 명시한다
- 직전 패스 대비 시각 크기를 1/3 수준으로 더 줄여 내비게이션과 균형을 맞춘다
- 내비게이션 수와 시각 장식을 유지하되, 헤더 높이와 간격을 함께 압축한다

### 시각 요구사항

- 카드의 라운드와 패딩을 한 단계 더 줄인다
- 라이브러리 카드는 3열보다 안정적인 2열 위계를 우선 검토한다
- 본문 패널 수는 유지하되, 카드 내부 정보 구조를 더 짧게 정돈한다
- 배경 이펙트와 glass는 포인트 정도만 남긴다

### 콘텐츠 요구사항

- Hero는 짧은 제목 + 짧은 설명 + 최소 행동으로 유지한다
- Hero proof와 library overview의 카운트 및 설명 문구를 13개 프로젝트 기준으로 고친다
- 라이브러리 카드의 노출 정보는 제목, 한 줄 문제, 대표 근거, 액션 정도로 압축한다
- About 페이지 문장은 소개서처럼 읽히게 하고, 대시보드처럼 보이지 않게 한다

### 인터랙션 요구사항

- 프로젝트 상세 모달 유지
- 필터는 유지하더라도 수와 시각 강도를 줄인다
- `prefers-reduced-motion` 대응 유지
