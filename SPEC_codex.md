# SPEC_codex

## 목표 사용자

- Blue Verse를 처음 보는 잠재 고객
- 복잡한 운영 구조를 가진 서비스 구축 파트너를 찾는 의사결정자
- 사례를 빠르게 훑고 신뢰 판단을 내리고 싶은 방문자

## 사용자 스토리

- 방문자는 첫 화면에서 이 팀이 "무슨 분야"보다 "어떤 복잡도"에 강한지 바로 이해해야 한다.
- 방문자는 대표 사례 3건만 먼저 읽어도 대략적인 신뢰 판단이 가능해야 한다.
- 방문자는 전체 프로젝트 목록에서 필요한 사례를 부담 없이 비교할 수 있어야 한다.
- 방문자는 About 페이지에서 팀의 태도와 작업 방식을 더 차분하게 확인할 수 있어야 한다.

## 핵심 문제

- 현재 사이트는 섹션과 패널 수가 많아 브랜드보다 UI 장식이 먼저 보인다.
- CTA, 칩, 작은 지표 상자가 너무 많아 한 번에 처리해야 할 정보가 많다.
- 헤더의 브랜드 존재감이 약하고, 대신 작은 캡슐형 내비게이션이 시선을 가져간다.
- 카피가 다소 설명 과잉이라 신뢰감보다 피로감을 줄 수 있다.

## 리디자인 요구사항

### 정보구조

#### index.html

1. Quiet Hero
2. Featured Cases
3. Project Library
4. Compact Method Strip
5. CTA / Footer

#### about.html

1. About Hero
2. Positioning Statement
3. Process
4. Coverage
5. CTA / Footer

### 헤더 요구사항

- 헤더 로고는 실제 PNG 워드마크를 사용한다
- HTML 이미지 속성에는 실제 자산 치수(593x117)를 명시한다
- 데스크톱에서 현재보다 훨씬 크게 보이게 조정한다
- 내비게이션 수와 시각 장식을 줄여 로고가 주인공이 되게 한다

### 시각 요구사항

- 박스 안의 박스를 줄인다
- 3열 카드보다는 2열 혹은 1열 위계를 우선한다
- 본문 패널 수를 줄이는 대신 여백과 타이포 위계로 구분한다
- 배경 이펙트와 glass는 포인트 정도만 남긴다

### 콘텐츠 요구사항

- Hero는 짧은 제목 + 짧은 설명 + 최소 행동으로 정리한다
- CTA 문구는 덜 세일즈적이고 더 차분한 톤으로 바꾼다
- 라이브러리 카드의 노출 정보는 제목, 한 줄 문제, 대표 근거, 액션 정도로 압축한다
- About 페이지 문장은 소개서처럼 읽히게 하고, 대시보드처럼 보이지 않게 한다

### 인터랙션 요구사항

- 프로젝트 상세 모달 유지
- 필터는 유지하더라도 수와 시각 강도를 줄인다
- `prefers-reduced-motion` 대응 유지
