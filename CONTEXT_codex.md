# CONTEXT_codex

## 프로젝트 목적

Blue Verse 포트폴리오의 최근 리디자인을 한 번 더 다듬어, 카드 정렬 붕괴와 과한 여백 문제를 정리한다.
이번 패스의 핵심은 "더 화려하게"가 아니라 "더 정확하게 정돈된 정보 밀도"다.

## 이번 세션 범위

- 헤더 워드마크를 직전 상태 대비 1/3 수준으로 더 줄여 네비게이션과 균형을 맞춘다.
- 카드 라운드와 패딩, 섹션 간격을 더 줄여 전체 레이아웃을 조밀하고 안정적으로 만든다.
- Featured와 Project Library 카드 레이아웃이 깨져 보이지 않도록 구조를 다시 맞춘다.
- `core/docs/market-intel` 기준 13개 프로젝트가 모두 반영되어 있는지 검증하고 카피 숫자도 그 기준으로 맞춘다.
- `SUIT` 웹폰트를 유지한다.
- `hero_glass_mac.png`는 계속 사용하지 않는다.

## 현재 문제 인식

- 최근 declutter 패스 이후에도 Featured와 Library 카드가 여전히 넓고 느슨해 보여 카드 간 위계가 흐린다.
- 일부 카드가 3열 상태에서 답답하게 보여 "정리된 밀도"보다 "작아진 듯한 어수선함"으로 인식될 위험이 있다.
- 헤더 워드마크가 직전 패스 기준으로는 여전히 존재감이 강해 본문 시작 전에 무게가 과하게 실린다.
- Hero proof 카피가 여전히 `12개 사례`를 기준으로 남아 있어 현재 `13개 프로젝트` 데이터셋과 불일치한다.

## 리디자인 방향

### 구조 방향

- Home: `Quiet Hero -> Featured Cases -> Project Library -> Compact Method -> CTA`
- About: 기존 declutter 구조를 유지하되 여백과 카드 밀도만 함께 정돈한다.
- 카드 수는 유지하되 각 카드 내부 구조를 한 단계 더 단순화한다.

### 시각 방향

- 더 작은 라운드
- 더 짧은 패딩
- 더 짧은 수직 리듬
- 더 안정적인 카드 폭
- 로고보다 콘텐츠가 먼저 보이는 헤더

### 콘텐츠 방향

- 숫자와 표현은 반드시 현재 13개 프로젝트 기준으로 맞춘다.
- Hero와 overview 카피는 더 짧고 단정하게 유지한다.
- 카드는 "무엇을 했는가"보다 "어떤 문제를 어떻게 정리했는가"가 먼저 읽히게 한다.
